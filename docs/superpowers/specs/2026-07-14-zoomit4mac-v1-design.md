# ZoomIt4Mac v1 — Design

Date: 2026-07-14
Status: Approved (brainstorming complete)

## Goal

A native macOS re-implementation of Sysinternals ZoomIt, written in Swift. v1 ships the core trio: **Zoom** (frozen-screen magnification), **Draw** (on-screen annotation), and **Type** (text on the overlay), triggered by global hotkeys from a menu bar app with a settings window. Break timer, live zoom, and screen recording are explicitly out of scope for v1 and come later.

## Decisions

| Topic | Decision |
|---|---|
| v1 scope | Zoom + Draw + Type, global hotkeys, menu bar app, settings window |
| Minimum macOS | 14 (Sonoma) |
| Distribution | Developer ID signing + notarization (hardened runtime); not Mac App Store |
| Sandbox | Disabled (screen capture across all windows; future event-tap needs) |
| Default hotkeys | ZoomIt parity: ⌃1 Zoom, ⌃2 Draw; rebindable in settings |
| Project tooling | XcodeGen (`project.yml` in git, `.xcodeproj` generated, not committed) |
| Structure | Approach A: pure-Swift `ZoomItCore` framework + thin AppKit app shell |
| Tests | Swift Testing, core-only test target, headless, no TCC dependencies |
| UI frameworks | AppKit for overlay/zoom windows; SwiftUI for the settings window |

## Architecture

Three targets, generated by XcodeGen:

1. **`ZoomItCore.framework`** — pure Swift, no AppKit import. All logic and state.
2. **`ZoomIt4Mac.app`** — AppKit shell + SwiftUI settings. `LSUIElement = YES` (menu bar only, no Dock icon). Hardened runtime, Developer ID.
3. **`ZoomItCoreTests`** — Swift Testing bundle testing `ZoomItCore` only; no app host.

Event flow is one-directional: hotkey / keyboard / mouse events → `SessionStateMachine` → emitted effects → shell executes them (capture screen, show/dismiss overlays) → results (captured image, failures) fed back into the state machine as events.

### Core module components

- **`SessionStateMachine`** — single source of truth for the active mode: `.idle`, `.zoom(level:center:)`, `.draw(zoomed:)`, `.type(...)`. Only legal transitions allowed: ⌃2 or left-click while zoomed enters draw-on-zoom (ZoomIt behavior); Esc unwinds type → draw → zoom → idle; ⌃1 while zoomed exits zoom. Emits effects (`captureScreen`, `showOverlay`, `dismissOverlay`, …) consumed by the shell.
- **`ZoomGeometry`** — pure functions: zoom factor clamping (1×–8×), ZoomIt-style mouse-follow pan mapping (screen edges reachable at any zoom level), point/rect transforms between screen space and magnified-image space, per-display `backingScaleFactor` handling, multi-display arrangements including negative origins.
- **`AnnotationCanvas`** — ordered annotation model: freehand stroke, straight line, arrow, rectangle, ellipse; current color and pen width; unlimited undo stack within a session; erase-all; solid whiteboard/blackboard background fill.
- **`TypeTool`** — text runs with position, current color, live-adjustable font size, caret handling. Left-aligned only in v1.
- **`HotkeyConfiguration`** — `Codable` mapping of feature → key combo, duplicate/conflict detection, defaults ⌃1 / ⌃2.
- **`SettingsStore`** — `Codable` settings persisted via `UserDefaults` behind a protocol so tests use a fake store.

### Shell components (app target)

- **`HotkeyRegistrar`** — Carbon `RegisterEventHotKey` wrapper. Works without Accessibility permission. Behind a `HotkeyRegistering` protocol.
- **`ScreenSnapshotter`** — one-shot capture per display via `SCScreenshotManager` (ScreenCaptureKit). Behind a `Snapshotting` protocol.
- **`OverlayWindowController`** — one borderless `NSWindow` per display at `.screenSaver` level with `.canJoinAllSpaces` and `.fullScreenAuxiliary` collection behavior; hosts the snapshot layer (zoom via Core Animation transform) and the annotation view; local keyboard/mouse monitors feed events into the state machine.
- **`StatusItemController`** — menu bar icon and menu: Zoom, Draw, Settings…, About, Quit.
- **`SettingsWindow`** — SwiftUI: hotkey recorder, default zoom level, pen color/width defaults, launch at login via `SMAppService`.
- **`PermissionCoordinator`** — Screen Recording TCC detection and System Settings guidance.

## Feature behavior

### Zoom (⌃1)

- Trigger → freeze: one snapshot per display; overlay windows appear and smoothly animate from 1× to the default zoom level (2×, configurable) over ~250 ms with ease-out, centered on the mouse (ZoomIt-style smooth zoom).
- Zoom control: scroll wheel, trackpad pinch, or ↑/↓ arrows. Range 1×–8×, smooth stepping.
- Pan: mouse movement pans via the ZoomIt mouse-follow mapping (all screen edges reachable at every zoom level).
- Left-click enters draw mode on the frozen zoomed image. Right-click or Esc exits zoom and dismisses overlays. ⌃1 again also exits.
- Multi-display: every display freezes its own snapshot; zoom interaction happens on the display containing the mouse.
- The app's own overlay windows are excluded from capture (no feedback loops).

### Draw (⌃2 from idle, or left-click from zoom)

- Transparent full-screen overlay (or drawing onto the frozen zoom image), crosshair cursor.
- Pen: left-drag freehand. Modifier-drag shapes: Shift = straight line, Ctrl+Shift = arrow, Ctrl = rectangle, Tab held = ellipse.
- Colors via keys: R red, G green, B blue, O orange, Y yellow, P pink.
- Pen width: ⌘scroll adjusts.
- Undo: ⌘Z (unlimited within session); right-click also undoes last annotation (ZoomIt parity). E erases all. W fills solid white background (whiteboard), K solid black (blackboard).
- ⌘S saves the annotated screen as PNG via save panel; ⌘C copies it to the clipboard. The export is a screen capture of the annotated display — desktop, frozen zoom image, or board, with the ink included (overlay windows are capturable). When Screen Recording permission is missing, it degrades to compositing just the overlay content (ink over transparency in plain draw).
- Esc: back to zoom if draw was entered from zoom, otherwise exits to idle.
- Out of scope for v1: blur/highlighter tool (ZoomIt's X key).

### Type (T while in draw mode)

- Click sets the caret; typed text renders in the current pen color. ⌘+ / ⌘− adjust font size. Esc returns to draw mode. Empty text runs are discarded. Right-aligned typing (ZoomIt's ⇧T) deferred.

### Settings

Hotkey recorder for both features (with conflict highlighting), default zoom level, pen color/width defaults, launch at login.

## Error handling and permissions

- **Screen Recording permission**: checked lazily on first zoom trigger via an `SCShareableContent` fetch. If missing: non-blocking alert with an "Open System Settings" deep link (`x-apple.systempreferences:com.apple.preference.security?Privacy_ScreenCapture`), and zoom aborts cleanly back to `.idle`. Draw and Type on a transparent overlay require no permission, so the app is useful before permission is granted.
- **No Accessibility permission in v1**: Carbon hotkeys plus local event monitors inside our own key windows are sufficient.
- **Hotkey registration failure** (combo taken by another app): warning badge on the status item; the conflicting combo is highlighted red in Settings. Never a silent loss of a hotkey.
- **Snapshot failure** (ScreenCaptureKit error, display disconnected mid-capture): state machine receives `.captureFailed`, returns to `.idle`, signals audibly (beep) and logs; a richer transient notification is post-v1 polish. All SCK calls are `async throws` and funnel into one failure handler; no crash paths from capture errors.
- **Display topology change during an active mode**: on `NSApplication.didChangeScreenParametersNotification`, force-exit to `.idle` and dispose all overlays (simple and safe; no overlay migration).
- **Corrupt persisted settings**: fall back to defaults and log; never block launch.

## Testing strategy

- **Framework**: Swift Testing (`@Test`, `#expect`) in `ZoomItCoreTests`; runs headless via `xcodebuild test` with no TCC or display dependencies.
- **`SessionStateMachine`**: exhaustive transition-table tests — every (state, event) pair asserts the next state and emitted effects. Edge cases: ⌃1 while zoomed (exit), ⌃2 during zoom (draw-on-zoom), Esc unwind chains, `.captureFailed` mid-entry, display change mid-mode.
- **`ZoomGeometry`**: property-style tests — pan mapping reaches all screen corners at every zoom level; clamping at 1× and 8×; screen↔image point round-trips within epsilon; mixed 1×/2× scale factors; negative-origin display arrangements.
- **`AnnotationCanvas`**: undo/redo ordering, undo on empty stack is a no-op, erase-all then undo restores, color/width state across mode re-entry.
- **`TypeTool`**: caret positioning, font-size clamping, empty-run discard.
- **`HotkeyConfiguration`**: duplicate-combo detection, `Codable` round-trip, invalid stored data falls back to defaults.
- **`SettingsStore`**: fake defaults store; corrupt-data fallback.
- **Shell layer**: kept thin by design. `Snapshotting` and `HotkeyRegistering` fakes allow full mode-flow integration tests inside the core test target. AppKit window code is verified by running the app, not unit-tested.

## Build commands

```sh
brew install xcodegen          # once
xcodegen                       # regenerate ZoomIt4Mac.xcodeproj after project.yml or file-list changes
xcodebuild -project ZoomIt4Mac.xcodeproj -scheme ZoomIt4Mac build
xcodebuild -project ZoomIt4Mac.xcodeproj -scheme ZoomIt4Mac test
xcodebuild -project ZoomIt4Mac.xcodeproj -scheme ZoomIt4Mac test \
  -only-testing:ZoomItCoreTests/SomeSuite/someTest
```

`.xcodeproj` is generated output and stays out of git; `project.yml` is the source of truth.

## Deferred features (out of scope for v1, revisit later)

Each entry lists the ZoomIt behavior to mirror and the expected macOS implementation angle, so future versions can pick these up without re-deriving the research.

### Break Timer (⌃3)

Full-screen countdown with configurable duration, elapsed/remaining display, optional background image or blurred desktop, timer position options, and sound on expiry. Implementation: mostly independent of the zoom/draw machinery — a dedicated full-screen overlay window plus a `BreakTimer` model in `ZoomItCore` (start/pause/resume/expiry, formatting). Low risk; natural first post-v1 feature. Advanced ZoomIt options (show time of day when expired, run on specific display) can follow incrementally.

### Live Zoom (⌃4)

Magnified view where screen content keeps updating (video playback, demos) instead of a frozen snapshot. Implementation: continuous capture via ScreenCaptureKit `SCStream` rendering into the overlay's layer, with the same `ZoomGeometry` pan/zoom math. Must exclude our overlay windows from the stream to avoid feedback loops (`SCContentFilter` window exclusion). GPU/performance sensitive; needs frame-rate and latency tuning. On macOS the system "Hover Text"/Accessibility zoom overlaps somewhat — our version stays app-controlled.

### Screen Recording (⌃5)

Record full screen or zoomed region to a movie file, with optional microphone audio and click highlighting; ZoomIt ≥ v6 parity. Implementation: `SCStream` + `SCRecordingOutput` (macOS 15+) or `AVAssetWriter` fed by stream frames (macOS 14). Adds microphone TCC permission when audio enabled. Consider raising the deployment target to macOS 15 when this lands to use `SCRecordingOutput` directly.

### Snip / region screenshot (⌃6 in recent ZoomIt)

Rectangular region selection copied/saved as an image. Implementation: selection rectangle on the existing overlay window plus cropped snapshot; small feature once overlay plumbing exists.

### DemoType

Types a scripted text file automatically into the foreground app, simulating live typing (with speed control). Implementation: `CGEvent` keyboard synthesis — **requires Accessibility permission**, which v1 deliberately avoids. Adding this changes the permission story; gate it so the permission is only requested when the feature is first used.

### Blur / highlighter pens (X key and highlight mode in draw)

Blur tool obscures a dragged region; highlighter draws translucent wide strokes. Implementation: additional annotation types in `AnnotationCanvas` (translucent stroke is trivial; blur needs Core Image `CIGaussianBlur` applied to the underlying snapshot region — draw-on-zoom only, since a transparent overlay has nothing to blur).

### Right-aligned Type (⇧T)

Text grows leftward from the click point. Small addition to `TypeTool` (alignment property + layout).

### Screen-drawing niceties

ZoomIt extras worth batching into a draw-polish release: center-screen crosshair origin (drawing from center with modifier), degree/pixel readouts while drawing lines, copy-with-cursor in saved screenshots.

### Distribution expansion

Mac App Store is likely permanently out (sandbox incompatible with capture + future event synthesis). Homebrew cask (`brew install --cask zoomit4mac`) is the realistic reach channel once Developer ID notarization pipeline exists; add Sparkle for in-app updates when distributing outside the App Store.
